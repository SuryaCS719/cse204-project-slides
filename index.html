Cantor's Diagonalization

Are all infinities created equal?

Suryakiran Valavala
Vijay Arvind Ramamoorthy

The Nature of Sets

A set is just a collection of distinct objects.

123

We can count finite sets easily. But what about infinite ones?

Countable Infinity

A set is countable if you can list its elements in a sequence: 1st, 2nd, 3rd...

The Natural Numbers (ℕ) are the prototype for this.

N = { 1, 2, 3, 4, 5, ... }

The Bold Claim

Cantor proved that the Real Numbers between 0 and 1 are not countable.

No matter how you try to list them, you will always miss one.

The List

Let's pretend we can list all real numbers between 0 and 1.

We write them in binary expansion.

The Diagonal

Look at the diagonal bits (highlighted).

Task: Click the bits below to "flip" them and create a new number.

New Number: 0.

The Contradiction

This new number differs from the 1st number in the 1st bit.

It differs from the 2nd number in the 2nd bit.

It differs from the n-th number in the n-th bit.

Therefore, it is not on the list.

Formalizing the Argument

Let S = {x1, x2, x3...} be our list.

We define the diagonal number D = 0.d1d2d3... where:

dn = 1   if   (xn)n = 0
dn = 0   if   (xn)n = 1

Proof: For any n, the n-th bit of D differs from the n-th bit of xn

dn ≠ (xn)n ⟹ D ≠ xn

Since D is not equal to any xn, D is not in S.

Uncountable

We assumed we could list them all, but we constructed one that was missing.

Thus, the set of Real Numbers is Uncountable.

But what does this mean for computers?

Computers are Countable

Every computer program (or Turing Machine) can be written as a finite binary string.

Program = "01101001..." -> Integer

This means we can list every possible program: M1, M2, M3...

And if we have a list, we can try to diagonalize against it.

The Counting Argument

We just proved:

  • Set of Programs is Countable (Like ℕ)
  • Set of Problems is Uncountable (Like ℝ)
|Programs| < |Problems|

Conclusion: There are infinitely many problems that cannot be solved by any computer.

The Halting Grid

Imagine a grid where rows are Programs (M) and columns are Inputs (w).

Cell (i, j) = 1 if Mi halts on input wj, and 0 if it loops.

The "Nemesis" Machine

We build a new machine N that runs on input wi:

  1. Run machine Mi on input wi (the diagonal).
  2. If Mi halts, Loop forever.
  3. If Mi loops, Halt.

It does the opposite of the diagonal.

Formalizing the Paradox

Assume N exists. Since we listed all machines, N must be some Mk.

What happens when we run N on its own input wk?

N(wk) Halts   ⟺   Mk(wk) Loops
N(wk) Loops   ⟺   Mk(wk) Halts

But since N is Mk, we can substitute:

N(wk) Halts ⟺ N(wk) Loops

This is a logical impossibility (A ⟺ NOT A).

Undecidability

Is N on our list of all machines?

If N is some Mk, what happens when we run N(wk)?

It halts if and only if it loops.

Contradiction. The Halting Problem is Unsolvable.
